# 贡献指南

感谢你对 KeyHunter 项目的关注！我们欢迎任何形式的贡献，包括但不限于报告问题、提交代码、完善文档。

## 行为准则

请参阅 `CODE_OF_CONDUCT.md`，我们期望所有参与者遵守。

## 如何报告问题

1. 使用 GitHub Issues 提交，选择对应的模板（若有）。
2. 描述问题时请提供：
   - 操作系统和 Python 版本。
   - 复现步骤（命令、参数、目标文件内容示例）。
   - 预期行为和实际行为。
   - 如果可能，附上扫描输出或错误信息（请勿包含真实密钥）。

## 如何提交代码

1. Fork 本仓库到你的账号。
2. 创建新分支，分支名建议使用 `feature/xxx` 或 `fix/xxx` 格式。
3. 确保代码符合以下规范：
   - 遵循 PEP 8 风格。
   - 新功能必须包含适当的注释和文档字符串。
   - 确保所有现有功能不受影响（可通过手动测试）。
4. 提交前，请确保没有包含任何真实密钥或敏感数据。
5. 发起 Pull Request，描述本次修改的内容和目的。

## 代码结构说明

- `keyhunter.py`：命令行入口，负责解析参数，调用核心模块。
- `config/`：存放默认配置。
- `core/`：扫描引擎和规则库。
- `utils/`：报告生成。

## 扩展规则

如果你希望添加新的内置规则，请修改 `core/rules.py` 中的 `DEFAULT_PATTERNS`，并确保新规则不会导致过多误报。

## 测试

目前项目未集成自动化测试，但建议在提交前手动运行扫描示例目录，确认功能正常。

## 文档

若修改了功能或配置，请同步更新 README.md 和本文件。

## 联系方式

如有疑问，可通过 GitHub Issue 联系维护者。
